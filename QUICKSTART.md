# 快速开始指南

欢迎使用 **生图生视频工具**！这个简单的指南会帮你在 5 分钟内开始使用。

## ⚡ 5 分钟快速开始

### 步骤 1：启动服务器（1 分钟）

```bash
cd D:\Claude_Workspace
npm start
```

你会看到以下输出：
```
✓ 数据库初始化成功
✓ 服务器运行中: http://localhost:3000
✓ 轮询间隔: 30000ms
```

### 步骤 2：打开浏览器（30 秒）

在浏览器中访问：
```
http://localhost:3000
```

你会看到一个漂亮的紫色界面，顶部有三个选项卡：文生图、图生图、历史记录。

### 步骤 3：配置 API Token（30 秒）

1. 在顶部导航栏找到"输入 API Token"的输入框
2. 粘贴你的兔子 API Token（以 `sk-` 开头）
3. 点击"保存 Token"按钮
4. 看到绿色的"已保存 ✓"提示后，说明成功了

### 步骤 4：生成第一张图片（3 分钟）

#### 文生图方式：

1. **点击"文生图"选项卡**
2. **在大文本框输入图片描述**，例如：
   ```
   一只可爱的橙色小猫，坐在阳光洒满的窗边，
   背景是绿色的盆栽植物，画风温暖柔和，
   高质量，细节丰富，光影效果逼真
   ```
3. **点击"生成图片"按钮**
4. **等待结果**（通常 30 秒到 2 分钟）
5. **看到图片后，可以直接在页面上预览**

#### 图生图方式：

1. **点击"图生图"选项卡**
2. **输入参考图片的 URL**（可以是多张，用换行分隔）
   ```
   https://example.com/image1.jpg
   https://example.com/image2.jpg
   ```
3. **输入修改指示**，例如：
   ```
   把背景改成蓝色的天空，
   添加一些彩色的气球
   ```
4. **点击"生成图片"按钮**
5. **等待并查看结果**

## 📚 查看历史记录

点击"历史记录"选项卡，你会看到所有已生成的任务：

- ✅ **已完成的任务**：显示生成的图片、提示词和生成时间
- ⏳ **处理中的任务**：显示旋转的加载动画
- ❌ **失败的任务**：显示错误信息和"重试"按钮

### 在历史记录中的操作：

| 按钮 | 功能 |
|-----|------|
| 🗑️ 删除 | 删除历史记录 |
| 🔄 重试 | 重新提交失败的任务 |
| ⬇️ 下载 | 下载生成的图片 |

## 🎨 提示词编写技巧

好的提示词是生成高质量图片的关键。以下是一些建议：

### ✅ 好的提示词示例

```
【人物描述】
一个 25 岁左右的亚洲女孩，长黑直发，
穿着浅蓝色的棉麻连衣裙，脚踩白色运动鞋

【场景描述】
站在盛开的樱花树下，春风吹动樱花飘落，
明媚的阳光从树叶间洒下，形成漂亮的光影

【风格描述】
日式和风插画风格，温暖唯美，
色彩饱和度高，光影渲染细致
```

### ❌ 不好的提示词示例

```
一个女孩
漂亮的图片
好看的场景
```

### 提示词编写建议

1. **具体描写**：用形容词描绘细节，而不是抽象概念
2. **分段描述**：分别描写人物、场景、光线、风格
3. **参考风格**：可以提到艺术风格（油画、水彩、动画、摄影等）
4. **质量词汇**：加入"高质量""细节丰富""逼真"等词
5. **使用逗号或换行**：让模型更容易理解每个部分

## 🔧 常见问题

### Q: 生成一张图片需要多久？

A: 通常 30 秒到 3 分钟，取决于服务器负载。系统会每 30 秒自动检查一次。

### Q: 可以同时生成多张图片吗？

A: 可以！你可以在多个浏览器标签页中同时提交任务，系统会并发处理。

### Q: 生成的图片在哪里保存？

A:
- 前端页面中可以直接预览
- 历史记录中可以再次查看
- 可以通过"下载"按钮保存到本地电脑

### Q: 如何删除旧的历史记录？

A: 在历史记录页面，点击任何任务右侧的"删除"按钮即可。

### Q: Token 保存失败怎么办？

A:
1. 确保 Token 以 `sk-` 开头
2. 检查浏览器是否允许存储数据
3. 在浏览器控制台查看错误信息（F12 → Console）

### Q: 生成失败了怎么办？

A: 在历史记录中找到失败的任务，点击"重试"按钮重新提交。

## 🌐 在团队中使用

### 局域网共享

如果你想让公司同事也能使用这个工具：

1. **在服务器电脑上启动应用**
   ```bash
   npm start
   ```

2. **确保防火墙允许访问**
   - Windows：允许 Node.js 通过防火墙
   - 或者暂时关闭防火墙的特定端口

3. **其他同事在浏览器中访问**
   ```
   http://你的电脑IP:3000

   例如：http://192.168.1.100:3000
   ```

4. **每个人保存各自的 API Token**（如果 Token 不同的话）

## ⚙️ 自定义配置

如果你需要调整设置，编辑 `.env` 文件：

```env
# 改变服务器端口（如果 3000 被占用）
PORT=8080

# 改变轮询间隔（改成 10 秒）
POLL_INTERVAL=10000

# 改变任务超时时间（改成 10 分钟）
POLL_TIMEOUT=600000
```

修改后需要**重启服务器**才能生效。
