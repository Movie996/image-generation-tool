# 🎨 Image Generation Tool

> AI 图片生成工具 — 文生图 / 图生图，基于 nanobanana-2 模型（通过兔子 API）

![Node.js](https://img.shields.io/badge/Node.js-20.6+-green.svg)
![License](https://img.shields.io/badge/License-MIT-blue.svg)

## ✨ 功能特性

- 📝 **文生图 (Text-to-Image)** — 输入提示词，一键生成图片
- 🖼️ **图生图 (Image-to-Image)** — 粘贴图片 URL，根据提示词转换风格
- ⚡ **队列模式** — 提交即释放，连续提交不阻塞 UI
- 🔔 **即时反馈** — 居中弹窗显示提交成功（2 秒自动消失）
- 📊 **历史记录** — 完整任务管理：查看、下载、重试、删除
- 🔄 **智能刷新** — 增量更新防抖动，不打断页面操作
- 🖥️ **终端可见** — 服务器日志实时输出，方便排查问题

## 🚀 快速开始

### 前置要求

- [Node.js](https://nodejs.org/) >= 20.6（推荐最新 LTS v22.x）
- 兔子 API Token（[获取方式](#%EF%B8%8F-配置说明)）

### Windows 用户（推荐）

**双击 `启动工具.bat` 即可，全程自动。**

脚本自动完成：
1. ✅ 检测 Node.js，未安装时用 `winget` 自动安装
2. ✅ 校验 Node.js 版本是否满足 ≥ v20.6
3. ✅ 首次运行自动执行 `npm install` 安装依赖
4. ✅ 启动服务器，终端窗口保持可见，日志实时输出
5. ✅ 2 秒后自动打开浏览器

> 关闭终端窗口即可停止服务，无需停止脚本。

### Mac / Linux 用户

首次使用前，给脚本加执行权限（只需一次）：

```bash
chmod +x 启动工具.sh
```

之后每次双击或在终端运行：

```bash
./启动工具.sh
```

### 手动启动（所有平台）

```bash
# 1. 安装依赖
npm install

# 2. 配置环境变量（编辑 .env 填入你的 Token）
cp .env.example .env

# 3. 启动
npm start
```

## ⚙️ 配置说明

### 获取 API Token

1. 访问 [兔子 API](https://api.tu-zi.com) 平台
2. 注册账号 → 登录控制台
3. 复制你的 API 密钥（以 `sk-` 开头）

### 配置 .env 文件

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `TUZI_API_TOKEN` | 你的兔子 API 密钥 | **必填** |
| `TUZI_API_BASE_URL` | API 地址 | `https://api.tu-zi.com/v1` |
| `PORT` | 服务端口 | `3000` |
| `POLL_INTERVAL` | 后端轮询间隔（毫秒） | `10000`（10 秒） |
| `POLL_TIMEOUT` | 任务超时时间（毫秒） | `600000`（10 分钟） |

> ⚠️ `.env` 文件包含密钥，**不要提交到 Git**（已在 `.gitignore` 中排除）

## 📖 使用指南

### 文生图

1. 打开浏览器访问 http://localhost:3000
2. 点击「文生图」选项卡
3. 输入详细描述（支持中英文，最多 2000 字符）
4. 点击 **生成图片**，看到成功弹窗后可继续提交
5. 切换到「历史记录」查看进度和结果

**提示词示例：**
```
一只猪在草地上玩耍，动漫风格，
阳光明媚，蓝天白云，草地绿油油的，
可爱治愈系画风，高质量，细节丰富
```

### 图生图

1. 切换到「图生图」选项卡
2. 粘贴图片 URL（多张换行分隔）
3. 输入修改描述（如：「转为动漫风格」）
4. 点击 **生成图片**

### 历史记录管理

| 操作 | 说明 |
|------|------|
| 👁 查看状态 | pending / completed / failed 实时显示 |
| 📥 下载 | 任务完成后点击下载按钮 |
| 🔁 重试 | 失败的任务一键重新提交 |
| 🗑 删除 | 单个删除或一键清空 |
| 🔍 放大 | 点击结果图片查看大图 |

### 连续批量提交

- 提交后**立即释放按钮**，无需等待返回
- 可以连续点击多次，每次弹出成功提示
- 所有任务进入队列，在历史记录中跟踪状态
- 后台每 10 秒轮询一次 API 更新状态

## 🐛 常见问题

### Q: 启动后浏览器打不开？

确认端口是否被占用：
```bash
# Windows：查看端口 3000 是否被使用
netstat -ano | findstr :3000

# 如被占用，修改 .env 中的 PORT 为其他端口
```

### Q: 提示 "Failed to fetch"

1. 确认服务已正常启动（终端无报错）
2. 确认浏览器地址的端口与 `.env` 中 `PORT` 一致
3. 关闭终端重新双击 `启动工具.bat` 重启

### Q: 任务一直失败 / TLS 错误

- 确保网络可以访问 `api.tu-zi.com`（国内可能需要代理）
- 检查 API Token 是否有效且未过期
- 已设置 600 秒超时，基本可以覆盖大多数情况

### Q: Node.js 版本不满足要求？

下载最新 LTS 版本：https://nodejs.org/zh-cn/download  
（当前推荐 v22.x，最低要求 v20.6）

### Q: 如何修改端口？

编辑 `.env` 文件中的 `PORT=3000` 为你想要的端口号，重启服务后生效。
